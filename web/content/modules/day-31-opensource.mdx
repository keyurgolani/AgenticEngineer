## 31.10 Contributing to Open Source

### How to Start

1. **Find a project** that aligns with your interests
2. **Read the contributing guide** (CONTRIBUTING.md)
3. **Start with documentation** or small bug fixes
4. **Join the community** (Discord, Slack, GitHub Discussions)
5. **Be patient and persistent**

### High-Impact Contributions

| Type          | Impact                         |
| :------------ | :----------------------------- |
| Documentation | High - helps everyone          |
| Bug fixes     | Medium - improves stability    |
| New features  | High - but requires discussion |
| Tests         | Medium - improves reliability  |
| Examples      | High - helps adoption          |

### Projects Seeking Contributors

- **LangChain**: Always needs documentation and examples
- **CrewAI**: Growing fast, needs tools and integrations
- **MCP Servers**: Many services need MCP servers
- **Evaluation**: Benchmarks need more test cases
